MAIN GOAL
---------

To have an EXCEL document that lists all the network devices in our LAN / CAMPUS network (in a scenario where a network controller like DNAC is not possible to deploy) and their IOS version (current and optimal). We will leverage the device's REST API to check the current version and then compare it against the recommended one.

Objective 3 COMPLETE
-----------
* How to cancel all jobs and remove the malcopied file from the flash:
The idea — when the user hits cancel, you send clear line vty * to each device via a separate SSH connection (using paramiko or netmiko), which terminates the active SCP session on the device side, causing scp.put() to fail with an exception on the Python side. That exception is caught by the "except Exception as e block" in "transfer_ios_image" and returns 'TRANSFER_ERROR', cleanly exiting the thread.

The only risk is that clear line vty * will also kill any other active VTY sessions on those devices — not just the SCP one. In a lab that's probably fine, but worth being aware of in production.
